# Sync Workflow Reference

## Source Intake

For each source, extract:

- signal: what new observation, phrase, decision, or tension appeared
- node impact: which existing node changes
- new node candidate: what deserves its own page
- link candidates: what should be connected
- outcome: public content, private material, product judgment, course, Skill, or review

## Node Update Pattern

1. Read nearby existing pages first.
2. Merge with the closest existing node when possible.
3. Create a new page only when the concept has its own durable role.
4. Add source links in frontmatter or body.
5. Update `Next Step` with a concrete small action.

## Gap Review

Look for:

- important nodes with no deliverable path
- repeated ideas that have no method page
- public signals that have no synthesis
- private material that should stay private but can inspire a public mechanism
- nodes with many links but no conclusion
- stale nodes with no recent decision

Gap shape:

```markdown
### [[Gap title]]
- What is missing:
- Why it matters:
- Needed judgment:
- Next step:
```

## Surprising Connections

